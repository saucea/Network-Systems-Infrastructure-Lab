Self-Hosted Ticketing System (Zammad)

Deployed and configured a self-hosted ticketing system using Zammad in a Docker container to manage and track electronics repair workflows.

Key Features:
+ Centralized ticket management for repair requests and job tracking
+ Organized customer issues, statuses, and resolutions in a structured system
+ Containerized deployment using Docker for portability and ease of maintenance
+ Persistent storage configuration to retain ticket data across container restarts

Use Case:
+ Track incoming repair jobs and their progress
+ Maintain history of completed repairs and customer interactions
+ Improve organization and workflow efficiency for a small electronics business

Skills Demonstrated:
+ Docker container deployment and management
+ Self-hosted application setup
+ Workflow automation and system design
+ Data persistence and service reliability
